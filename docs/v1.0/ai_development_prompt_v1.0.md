- **アプリ名:** はじめてメモ ~First Steps~
- **コンセプト:** 「初めての瞬間」だけをシンプルに記録・共有する育児マイルストーン記録アプリ。

## 2. デザインシステムの実装

まず、アプリ全体のUIの基礎となるデザインシステムを実装してください。以下の仕様に厳密に従ってください。

- **UIキット/コンポーネントライブラリ:** Material Design (Flutter標準) をベースに、以下のカスタマイズを適用します。
- **カラーパレット:**
  - `primaryColor`: `#2C3E50` (ダークネイビー)
  - `accentColor` / `colorScheme.secondary`: `#87CEEB` (ベビーブルー)
  - `scaffoldBackgroundColor`: `#FFFFFF` (ホワイト)
  - `cardColor`: `#F5F5F5` (ライトグレー)
  - `textTheme.bodyText1.color` / `textTheme.bodyText2.color`: `#2C3E50`
  - `errorColor`: `#B00020`
  - `indicatorColor` (for completed items): `#4CAF50` (グリーン)
- **タイポグラフィ:**
  - `fontFamily`: `Noto Sans JP` (Google Fontsから取得)
  - テキストのテーマ (`textTheme`) を定義し、見出し(H1, H2)、本文、キャプションなどのスタイルを統一してください。
- **コンポーネントスタイル:**
  - `ElevatedButton`, `Card`, `TextField` などの主要ウィジェットは、角丸 (`borderRadius: BorderRadius.circular(12.0)` など) を基本とします。
  - アイコンは `Icons` クラスの標準的なラインアイコンを使用します。

## 3. 開発ステップ

以下の順序で開発を進めてください。各ステップでコンポーネントをウィジェットとして適切に分割し、再利用性を高めてください。

### Step 1: プロジェクトセットアップと基本構造

1.  `flutter create first_steps` でプロジェクトを新規作成します。
2.  必要なパッケージを `pubspec.yaml` に追加します。
    - `google_fonts`
    - `hive` / `hive_flutter`
    - `path_provider`
    - `provider` (状態管理)
    - `intl` (日付フォーマット用)
3.  `main.dart` を設定し、上記で定義したデザインシステム（`ThemeData`）を適用します。
4.  ボトムナビゲーションバーを持つメインのScaffold (`MainScreen`) を作成します。タブは「ホーム」「タイムライン」「マイルストーン」「設定」の4つとします。（共有は機能実装時に追加）
5.  状態管理のために `provider` をセットアップします。

### Step 2: ローカルデータベース (Hive) のセットアップ

1.  子供のプロフィール情報 (`ChildProfile`) とマイルストーンの記録 (`MilestoneRecord`) を保存するためのHive Boxを初期化するコードを記述します。
2.  対応するデータモデルクラス（`TypeAdapter`を継承）を作成してください。

### Step 3: 画面と機能の実装 (MVP)

1.  **子供のプロフィール登録画面 (初回起動時):**
    - 名前、生年月日、写真（任意）を入力し、Hiveに保存する画面を作成します。

2.  **ホーム画面 (`HomeScreen`):**
    - Hiveからプロフィール情報を読み込み、名前と月齢を表示します。
    - Hiveから最新の記録3件を読み込み、カード形式で表示します。
    - 「記録する」ボタンを配置します。

3.  **マイルストーン一覧画面 (`MilestonesScreen`):**
    - 事前に定義されたマイルストーンのリスト（ハードコードでOK）を月齢ごとにグループ化して表示します。
    - Hiveに保存された記録と照合し、達成済みの項目にはチェックマークと日付を表示します。
    - 未達成の項目をタップすると、記録画面に遷移します。

4.  **記録画面 (`RecordScreen`):**
    - マイルストーン名、日付ピッカー、写真ピッカー（`image_picker`を使用）、メモ入力フィールドを配置します。
    - 「保存」ボタンをタップすると、入力内容をHiveに `MilestoneRecord` として保存します。

5.  **タイムライン画面 (`TimelineScreen`):**
    - Hiveから全ての記録を読み込み、日付の降順でカードリストとして表示します。

6.  **設定画面 (`SettingsScreen`):**
    - 現時点では、Pro版購入への導線（ダミーボタンでOK）、利用規約・プライバシーポリシー表示用のシンプルな画面へのリンクのみを配置します。

### Step 4: 共有機能の実装

1.  `screenshot` や `share_plus` パッケージを利用します。
2.  タイムラインの各カードに共有ボタンを設置します。
3.  ボタンをタップすると、そのカードに対応するウィジェットを画像に変換し、OSの共有シートを呼び出して共有できるようにします。
