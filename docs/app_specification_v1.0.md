# 育児マイルストーン記録アプリ アプリ仕様書

## 1. 概要

| 項目 | 内容 |
|:---|:---|
| **アプリ名** | はじめてメモ ~First Steps~ |
| **コンセプト** | 「初めての瞬間」だけを、シンプルに、美しく記録する。遠くの家族とも簡単に共有できる、ミニマルな育児マイルストーン記録アプリ。 |
| **ターゲット** | - **主要ターゲット:** 初めての子供を持つ、デジタルネイティブな親（20〜35歳）。多機能なアプリは続かないが、成長の節目は残したいと考えている層。<br>- **セカンダリターゲット:** 孫の成長を見守りたい祖父母。 |
| **解決する課題** | - 既存の育児記録アプリは機能が多すぎて入力が負担になる。<br>- 「初めて笑った日」のような特別な瞬間だけを記録し、後から見返したいというニーズ。<br>- 遠方の祖父母へ、子供の成長を手軽に共有したい。 |

## 3. 機能仕様 (MVP: v1.0)

### 3.1. コア機能（無料）

| 機能 | 詳細 |
|:---|:---|
| **子供のプロフィール登録** | - 登録できる子供は1人まで。<br>- 登録項目：名前（ニックネーム可）、生年月日、プロフィール写真（任意）。<br>- ホーム画面上部に常に表示され、現在の月齢が自動計算される。 |
| **マイルストーン一覧** | - 厚労省の資料などを参考に、月齢ごとに標準的なマイルストーンをテンプレートとして用意。<br>- 例：0-3ヶ月「初めての笑顔」「首すわり」、6-12ヶ月「お座り」「ハイハイ」「つかまり立ち」。<br>- 達成済みの項目と未達成の項目は視覚的に区別される（例：カラー表示 vs グレーアウト）。 |
| **マイルストーン記録** | - 一覧から項目をタップすると記録画面に遷移。<br>- 記録項目：達成日（カレンダー選択）、写真1枚（任意）、メモ（140字程度の短文、任意）。 |
| **タイムライン表示** | - 記録したマイルストーンを時系列で一覧表示。<br>- 各項目をタップすると詳細（写真・メモ）を閲覧・編集できる。 |
| **共有機能** | - タイムラインまたは記録詳細画面から、特定の記録を画像として生成。<br>- 生成された画像には、写真、マイルストーン名、日付、メモが含まれる。<br>- OSの共有シートを呼び出し、LINEやSNSなどで共有できる。 |

### 3.2. 課金機能（Pro版）

| 機能 | 価格案 | 詳細 |
|:---|:---:|:---|
| **広告の非表示** | ¥480（買い切り） | アプリ内の全ての広告を削除する。 |
| **複数人の子供登録** | Pro版に含む | 2人目以降の子供を登録・切り替えできるようになる。 |
| **PDF/画像一括出力** | Pro版に含む | 記録したマイルストーンをまとめて1つのPDFファイルや複数の画像として書き出し、印刷や保存に利用できる。 |
| **クラウドバックアップ** | Pro版に含む | 機種変更やアプリの再インストール時にデータを復元できるように、記録をクラウドにバックアップする。 |
| **カスタムマイルストーン** | Pro版に含む | テンプレートにない、独自の「初めて」を自由に追加できる。 |

### 3.3. 広告（無料版）

| 種類 | 配置場所 |
|:---|:---|
| **バナー広告** | ホーム画面、タイムライン画面、マイルストーン一覧画面の下部に控えめに表示。 |
| **インタースティシャル広告** | 記録を保存したタイミングで、3〜5回に1程度の低い頻度で表示。 |

## 4. 画面仕様

1.  **ホーム画面:** 子供のプロフィール、最近の記録3件、「記録する」ボタン、ボトムナビゲーションを配置。
2.  **マイルストーン一覧画面:** 月齢でグループ化されたマイルストーンリスト。達成状況がわかるように表示。
3.  **記録画面:** マイルストーン名、日付ピッカー、写真アップローダー、メモ入力フィールド、保存ボタンを配置。
4.  **タイムライン画面:** 記録済みのマイルストーンを時系列にカード形式でリスト表示。
5.  **設定画面:** Pro版購入への導線、利用規約、プライバシーポリシー、お問い合わせなどを配置。

## 5. 技術スタック（推奨）

| 要素 | 選択 | 理由 |
|:---|:---|:---|
| **フレームワーク** | Flutter | iOS/Androidのクロスプラットフォーム開発により、開発コストと時間を削減。 |
| **ローカルDB** | Hive (Dart製) | 高速なKey-Valueストア。シンプルなデータ構造に適している。 |
| **広告** | Google AdMob | Flutterとの親和性が高く、導入が容易。 |
| **アプリ内課金** | RevenueCat | iOS/Androidの課金処理を抽象化し、実装を簡素化。 |
| **バックアップ (Pro)** | Firebase (Authentication, Firestore, Storage) | ユーザー認証とデータ/画像のバックアップ基盤として堅牢。 |
